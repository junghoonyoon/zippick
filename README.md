# 부동산 유튜브 요약 MVP

`지금사도될까요?`의 유튜브 자막 수집/분석 흐름을 부동산 도메인으로 옮긴 별도 작업본이다.

## 현재 방향

서비스의 첫 진입점은 `내 조건의 매수 후보 3곳`이다. 자기자금, 차주 소득·부채,
예상 금리·기간, 검토 지역, 매수 목적, 입주 계획을 입력하면 구매력과 조건을
기준으로 비교 후보를 좁히고, 후보별 검토 이유와 결정 전 확인할 점을 보여준다.
검토 지역·매수 목적은 복수 선택할 수 있다. 면적·세대수·연식은 후보가 모두 통과해야 하는 단일 임계값이다.
매수 가능 상한은 DSR·LTV·주택가격별 대출상한과 부대비용을 반영해 자동 계산한다.
가격 후보는 별도 예산 입력 없이 구매 가능 상한에 가까운 순으로 우선 비교한다.
후보 중 2~3곳을 비교함에 담으면 가격, 예산 차이, 조건 적합도, 단지 규모,
가격 근거, 주의점과 다음 확인 항목을 한 화면에서 나란히 볼 수 있다.
각 후보 카드의 `네이버 부동산에서 보기` 링크는 네이버 단지번호가 확인된 경우
해당 단지 페이지를 같은 탭에서 바로 열고, 확인되지 않은 경우 단지명 검색 결과로 연결한다.
네이버 확인 후 브라우저 뒤로가기를 누르면 직전에 보던 후보 결과와 스크롤 위치로 복귀한다.

보유 주택 수, 생애최초 여부, 자기자금, 차주 소득과 기존 대출 상환액을 입력하면
현재 시행 중인 규제지역·LTV·주택가격별 주담대 상한·전입 의무를 후보별로
판정한다. 정책 결과는 기준일과 정부 원문을 함께 표시하며, DSR과 실제 대출
한도는 금융회사 심사가 필요한 참고값으로 구분한다.

서울 중저가 후보는 국토교통부 실거래가 공개시스템에서 내려받은 최근 거래를
단지별로 집계한 `data/seoul_small_apartment_price_bands.csv`를 사용한다. 현재
전용 50~60㎡, 10억원 이하, 거래 표본 3건 이상인 가격대 656개가 포함되어 있다.
취소 거래는 제외하며 가격 범위는 단지별 거래의 10~90백분위를 쓴다.

공식 CSV를 새로 받은 뒤 아래 명령으로 가격대를 갱신할 수 있다.

```bash
PYTHONPATH=pipeline python3 pipeline/import_molit_price_bands.py /path/to/official.csv
```

후보 단지를 선택하면 최근 부동산 유튜브 영상에서 해당 키워드를 언급한 영상을
찾고, 유튜버별 대표 영상 1개를 AI로 요약한다. 유튜브는 추천 자체가 아니라 판단
근거를 확인하는 단계로 사용한다.

의견 라벨은 `상승기대`, `관망`, `주의`, `단순언급`으로 나눈다.

- `상승기대`: 가격 상승, 수요 증가, 입지 개선, 개발 호재를 명시
- `관망`: 가격 부담, 금리, 공급, 거래량 확인 필요를 명시
- `주의`: 하락 가능성, 미분양, 공급 과잉, 전세 리스크, 고평가 경고
- `단순언급`: 지역/단지/정책 이름만 나오고 판단 없음

## 실행

부모 프로젝트의 Python 가상환경과 `설정.txt`를 그대로 쓴다.

```bash
./부동산검색실행.command
```

브라우저 주소:

```text
http://127.0.0.1:8766
```

## 설정

우선 부모 폴더의 `beaver-v2/설정.txt`를 읽고, 필요하면 `beaver-v2/부동산/설정.txt`로 덮어쓸 수 있다.

필수:

- `유튜브키`
- `분석방식`
- `오픈라우터키` 또는 `제미나이키` 또는 로컬 Ollama

선택:

- `공공데이터키`: 국토교통부_아파트 매매 실거래가 상세 자료 API 인증키. 있으면 예산 후보의 가격대를 최근 실거래가로 우선 보강하고, 없거나 권한 오류가 나면 `data/apartment_price_bands.csv`의 보강 가격대로 동작한다.

## 폴더 구조

| 경로 | 역할 |
|---|---|
| `앱화면/real-estate-search.html` | 부동산 검색 화면 |
| `pipeline/config.py` | 부동산 앱 설정과 채널 로스터 |
| `pipeline/real_estate_search.py` | 검색 인덱스, fallback 검색, 의견 집계 |
| `pipeline/analyze_real_estate.py` | 부동산 의견 분석 프롬프트와 LLM provider |
| `pipeline/budget_candidates.py` | 예산·필수 조건 기준 후보 선정 |
| `pipeline/policy_evaluator.py` | 사용자·후보별 현재 주택 정책 판정 |
| `pipeline/search_server.py` | 로컬 API 서버 |
| `data/housing_policy_snapshot.json` | 시행 중인 정책 기준과 공식 출처 스냅샷 |
| `pipeline/cache/` | 부동산 전용 검색/분석 캐시 |
